## 1. 하이퍼스레딩

하이퍼스레딩은(Hyper-Threading) CPU 코어 1개를 2개의 코어처럼 사용하는 기술이다.  
하이퍼스레딩은 CPU 코어 하나에 레지스터를 2개 세트로 장착하여 구현된다.  
- 물리적 코어: 실제로 계산을 수행하는 코어 연산유닛 1개
- 논리적 코어: 레지스터 세트 2개

레지스터가 2세트이기 때문에 코어 하나에 2개 스레드의 정보(프로그램카운터, 스택포인터 등)가 동시에 저장될 수 있다.  
그런데 코어는 여전히 1개인데 어떻게 2개 스레드가 동시에 돌 수 있을까?  

CPU 코어에는 여러가지 연산 유닛들이 있다.  
- 정수 스칼라 유닛(ALU, Arithmetic Logic Unit)
  - 산술 연산: 덧셈, 뺄셈, 곱셈, 나눗셈 (정수 범위)
  - 논리 연산: AND, OR, XOR, NOT 등
  - ALU는 코어 1개에 여러개가 들어있다.
- 부동소수점 스칼라 유닛(FPU, Floating Point Unit)
  - 실수 계산을 담당
- 주소 생성 유닛(AGU, Address Generation Unit)
  - 데이터를 읽거나 쓸 때 메모리의 물리 주소를 계산함
- 벡터 연산 유닛(SIMD, Vector Units)
  - 하나의 명령어로 여러개의 데이터를 동시에 처리함(예: 8개의 숫자를 동시에 더함)
  - MMX, SSE, AVX 등

1개 스레드가 실행될 때 CPU 코어의 연산유닛들을 100% 사용하는것은 아니다.  
스레드는 메모리에서 데이터를 가져오기를 기다리고 있거나, 이전 연산이 끝나기를 기다리고 있을 수 있다.  
하이퍼스레딩은 사용하지 않는 연산유닛을 다른 스레드가 사용하게 하여 2개 스레드가 마치 동시에 실행되는 것처럼 보이게 한다.

하지만 어쨋든 코어는 1개이기 때문에 2개 스레드가 완벽하게 동시에 실행되는것은 아니다.  
하이퍼스레딩은 싱글스레딩 코어와 비교했을 때 20%~30%의 성능 향상이 있다고 한다.
