# Chapter 1. Getting Started

## Chunks
Lua가 실행하는 각각의 코드조각을 chunk라고 한다.  
chunk는 파일이나 1줄의 명령어일 수 있다. 예를들면 print("Hello Workd") 같은것 말이다.  
chunk의 크기는 아주 클 수도 있는데, Lua는 data-description 언어로도 사용되기 때문에 수 메가바이트 크기의 chunk도 있다.  

## 주석
주석은 -- 로 시작한다.  
여러줄의 주석은 --[[ 로 시작하고 --]] 로 끝난다.  
```lua
-- 주석

--[[
여러줄의
주석
--]]
```

## 전역변수
전역변수는 선언이 필요없다. 단지 사용하기만 하면 된다.  
```lua
b     -- 선언하지 않고 사용한 b는 초기값은 nil 이다.
b = 10
b     -- b는 10이다.
```

## 타입과 값
Lua는 dynamically-typed 언어이다. 그래서 변수의 타입을 선언할 필요가 없다.  
Lua에는 8개의 기본적인 type이 있다.  
nil, Boolean, number, string, userdata, function, thread, table 이다.  

type 함수는 값의 타입을 알려준다.  
```lua
type(nil)           --> nil
type(true)          --> boolean
type(10.4 * 3)      --> number
type("Hello world") --> string
type(io.stdin)      --> userdata
type(print)         --> function
type(type)          --> function
type({})            --> table
type(type(X))       --> string
```
userdata 타입은 임의의 C 데이터이다.  
userdata 타입에 대해서는 assignment와 equality test를 빼고는 정해진 연산이 없다.  
userdata는 Lua가 임베딩된 어플리케이션이나 C 라이브러리가 사용하는 임의의 타입을 표현하기위해 사용된다.  
예를들면 표준 I/O 라이브러리는 열린 파일을 나타내기 위해 userdata를 사용한다.  

변수는 정의된 타입이 없다. 모든 변수는 모든 타입의 값을 저장할 수 있다.
```lua
type(a)           --> nil ('a' is not initialized)
a = 10
type(a)           --> number
a = "a string!!"
type(a)           --> string
a = nil
type(a)           --> nil
```

## Nil
nil은 값이 없음을 나타내는데 사용된다.  
초기화되지 않은 처음 사용하는 변수는 nil 값이 입력되어 있다.  
만약 전역변수는 제거하고 싶다면 전역변수에 nil을 할당하면 된다.
```lua
a = nil    -- a 변수는 제거된다.
```

## Booleans
`true`, `false`는 Booleans 타입이다.  
Lua의 조건 검사는 `false` 와 `nil` 을 거짓으로 취급한다.  
나머지 모든 값은 참으로 취급한다.  
예를들면 0 과 빈 문자열도 참으로 취급한다.  

Lua는 `and`, `or`, `not` 조건연산자를 제공한다.  
`and` 연산자의 결과는, and의 첫번째 피연산자가 false라면 첫번째 피연산자, 그리고 첫번째 피연산자가 true라면 두번째 피연산자이다.  
`or` 연산자의 결과는, or의 첫번째 피연산자가 true라면 첫번째 피연산자, 그리고 첫번째 피연산자가 false라면 두번째 피연산자이다.  
```lua
4 and 5         --> 첫번째 피연산자가 참   --> 5
nil and 13      --> 첫번째 피연산자가 거짓 --> nil
false and 13    --> 첫번째 피연산자가 거짓 --> false
0 or 5          --> 첫번째 피연산자가 참   --> 0
false or "hi"   --> 첫번째 피연산자가 거짓 --> "hi"
nil or false    --> 첫번째 피연산자가 거짓 --> false
```

`and` 와 `or` 연산자 모두 short-circuit evaluation을 사용하기 때문에, 두번째 피연산자는 오직 첫번째 피연산자가 부정 일때만 연산된다.  

Lua 에는 자주 사용되는 구문이 있다.  
다음은 x가 초기화되지 않았을 때 x에 v값을 할당한다.  
```lua
if not x then x = v end
```
(그런데 x에 false가 입력되지 않았어야 할 것이다.)

또다른 유용한 구문은 다음과 같다.  
```lua
a and b or c
```
이 구문은 C의 `a ? b : c` 와 동일하다.  

`not` 연산자의 결과값은 언제나 Boolean 타입이다.  
```lua
not nil       --> true
not false     --> true
not 0         --> false
not not 1     --> true
not not nil   --> false
```

# Chapter 3. Numbers

Lua 버전 5.2 까지는 모든 숫자가 double-precision floating-point 형태였다.  
버전 5.3 부터는 숫자 표현에 64-bit integer가 추가되었다.  

## Numerals
숫자는 여러 형태로 사용할 수 있다.
```lua
4         --> 4
0.4       --> 0.4
4.57e-3   --> 0.00457
0.3e12    --> 300000000000.0
5E+20     --> 5e+20
```

integer와 float 값은 모두 `number` 타입이다.
```lua
type(3)     --> number
type(3.5)   --> number
type(3.0)   --> number
```

Lua 에서는 같은 값의 integer와 float은 동일하다.  
```lua
1 == 1.0      --> true
-3 == -3.0    --> true
0.2e3 == 200  --> true
```

integer와 float을 구분해야 한다면 `math.type` 함수를 사용한다.  
```lua
math.type(3)     --> integer
math.type(3.0)   --> float
```

16진수는 맨 앞에 0x 를 붙이면 된다.
```lua
0xff      --> 255
0x1A3     --> 419
0x0.2     --> 0.125
0x1p-1    --> 0.5
0xa.bp2   --> 42.75
```

## The Mathematical Library
Lua는 표준 `math` 라이브러리를 제공한다.  
이 라이브러리에는 삼각함수(sin, cos, tan, asin 등), 로그함수, 반올림 함수, max, min, 난수발생 함수 등이 있다.  
그리고 pi, huge(=infinite) 등의 상수값도 제공한다.  
```lua
ath.sin(math.pi / 2)      --> 1.0
math.max(10.4, 7, -3, 20) --> 20
math.huge                 --> inf
```

## Random-number generator
`math.random` 함수는 의사 난수를 생성한다.  
```lua
math.randomseed(os.time())   -- 시드 설정
math.random()                -- 0 이상 1 미만의 float 난수 생성
math.random(6)               -- 1 이상 6 이하의 integer 난수 생성
math.random(0,5)             -- 0 이상 5 이하의 integer 난수 생성
```

## Conversions
숫자를 float으로 변환하려면 0.0을 더하면 된다.  
```lua
-3 + 0.0                  --> -3.0
0x7fffffffffffffff + 0.0  --> 9.2233720368548e+18
```

숫자를 integer로 변환하려면 숫자에 rounding 함수를 사용한다.(math.ceil, math.floor, math.modf),  
```lua
math.ceil(3.3)    --> 4
math.ceil(-3.3)   --> -3
math.modf(3.3)    --> 3 0.3
math.modf(-3.3)   --> -3 -0.3
```
또는 math.tointeger 함수를 사용한다. math.tointeger 함수는 값에 소수 부분이 없어야 한다.  
```lua
math.tointeger(-258.0)      --> -258
math.tointeger(2^30)        --> 1073741824
math.tointeger(5.01)        --> nil (not an integral value)
math.tointeger(2^64)        --> nil (out of range)
```

# Chapter 5. Tables

Table은 Lua의 핵심 데이터 구조이다.  
Lua에서는 array, set, record, 그리고 다른 많은 데이터 형태들에 table을 사용한다.  
또한 package와 object를 표현하는데도 table을 사용한다.
예를들면 `math.sin` 함수를 생각해보면, math 라이브러리의 sin 함수라고 생각하겠지만, Lua에서는 math table에서 Key가 "sin"인 value이다.  

Lua의 table은 본질적으로 array이다. table은 숫자, 문자열 뿐만 아니라 Lua의 모든 값(nil 제외)을 index로 사용할 수 있는 array 이다.  

Lua의 table은 value도, variable도 아니다. table은 object이다.  

빈 table은 {} 표현으로 생성할 수 있다.
```lua
a = {}                -- create a table and assign its reference
k = "x"
a[k] = 10             -- new entry, with key="x" and value=10
a[20] = "great"       -- new entry, with key=20 and value="great"
a["x"]                --> 10
k = 20
a[k]                  --> "great"
a["x"] = a["x"] + 1   -- increments entry "x"
a["x"]                --> 11
```

table은 언제나 익명 객체이다. table 객체 자체와 table을 가르키는 변수명 사이에는 어떠한 고정된 관계도 없다.
```lua
a = {}
a["x"] = 10
b = a         -- 'b' refers to the same table as 'a'
b["x"]        --> 10
b["x"] = 20
a["x"]        --> 20
a = nil       -- only 'b' still refers to the table
b = nil       -- no references left to the table
```

## Table Indices
전역변수와 마찬가지로, table의 초기화되지 않은 요소는 초기값이 nil 이다.  
또한 요소에 nil을 할당하여 필드를 삭제할 수 있다.
```lua
a["y"]     --> nil
```

table의 요소에는 인덱스 말고 필드명 표현으로도 접근할 수 있다.  
예를들면 a.name 은 a["name"] 과 동일하다.  
```lua
a = {}         -- empty table
a.x = 10       -- same as a["x"] = 10
a.x --> 10     -- same as a["x"]
a.y --> nil    -- same as a["y"]
```

integer와 float 타입 index는 다음과 같다.
```lua
a = {}
a[2.0] = 10
a[2.1] = 20

a[2]      --> 10
a[2.1]    --> 20
```
2와 2.0은 동일한 key인데, table의 key가 숫자일 경우에는 만약 숫자값을 정수로 변환할 수 있다면 자동으로 변환되기 때문이다.  
정수로 변환될 수 없는 숫자값은 변환없이 그대로 사용된다.  

## Table Constructors
table은 다양한 방식으로 초기화할 수 있다.

{} 는 빈 table이다.

아래와 같이 초기화 한다면
```lua
days = {"Sunday", "Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday"}

print(days[4]) --> Wednesday
```
days[1] 은 Sunday, days[7] 은 Saturday 이다.  
이렇게 초기화할 경우 인덱스의 가장 처음 값이 0이 아니라 1인 것에 주의하자.  

또한 record-like table 형태의 초기화 방식도 제공한다.
```lua
a = {x = 10, y = 20}
```
이것은 아래와 동일하다.
```lua
a = {}
a.x = 10
a.y = 20
```

table 안에 table이 있는 형태의 초기화도 가능하다.
```lua
polyline = { color="blue", 
             thickness=2, 
             npoints=4,
             {x=0, y=0},      -- polyline[1]
             {x=-10, y=0},    -- polyline[2]
             {x=-10, y=1},    -- polyline[3]
             {x=0, y=1}       -- polyline[4]
           }
```

## Arrays, Lists, and Sequences
전통적인 array도 table로 만든다.  
table이 순서대로 나열된 정수값 key를 가지고 있다면(예: 1,2,3,4,5), 이것은 array 이다.  

array에 대해 생각해보자.  
array를 사용하려면 array의 길이를 알아야 한다.  
그리고 array는 시작 인덱스부터 마지막 인덱스까지 중간에 비어있는 곳이 없다.  
그런데 table은 이러한 규칙에는 완전히 위배된다.  
길이 라는것도 없고, 중간 인덱스가 없을수도 있기 때문이다.  

Lua에서는 table을 array처럼 사용하기 위해 sequence 라는것을 정의한다.  
sequence는 "1부터 양의 정수 n까지의 인덱스에 모두 nil이 아닌 value가 존재하는 리스트" 이다.  
특정 인덱스의 value가 nil라는 것은 존재하지 않는 인덱스이다.
그리고 Lua에서는 sequence의 길이를 알기 위해서 # 연산자를 제공한다.  
\# 연산자는 table 내의 sequence의 길이를 반환한다.  
아래 코드는 인덱스 1부터 마지막 인덱스(#a)까지 table내의 sequence를 순회하며 print 한다.  
```lua
-- print the lines, from 1 to #a
for i = 1, #a do
  print(a[i])
end
```

\# 연산자는 다음과 같이 sequence의 맨 뒤에 요소를 삽입할때도 사용된다.
```lua
a[#a + 1] = v   -- appends 'v' to the end of the sequence
```
만약 table에 양의정수 key를 가지는 요소가 없다면 해당 table의 길이(#)는 0이다.

## Table Traversal
table을 key-value 형태로 순회하려면 `pairs` iterator를 사용한다.
```lua
t = {10, print, x = 12, k = "hi"}
for k, v in pairs(t) do
  print(k, v)
    --> 1 10
    --> k hi
    --> 2 function: 0x420610
    --> x 12
end
```
table의 요소들이 순회되는 순서는 알 수 없다. table의 요소들은 정렬되어 있지 않기 때문이다.  

table을 list로 순회하려면 `ipairs` iterator를 사용한다.
```lua
t = {10, print, 12, "hi"}
for k, v in ipairs(t) do
  print(k, v)
    --> 1 10
    --> 2 function: 0x420610
    --> 3 12
    --> 4 hi
end
```
이 경우에는 table의 요소를 순서대로 순회한다.

table을 sequence로 순회하려면 정수 인덱스를 사용한다.
```lua
t = {10, print, 12, "hi"}
for k = 1, #t do
  print(k, t[k])
    --> 1 10
    --> 2 function: 0x420610
    --> 3 12
    --> 4 h
end
```

## Safe Navigation
라이브러리의 함수가 존재하는지 확인한 다음 호출하려면 어떻게 해야할까?  
아래처럼 zipcode 함수를 호출하려고 하는데, 먼저 존재하는지 확인하고 호출하려고 한다.
```lua
zip = company.director.address.zipcode
```

이 때는 or 연산자로 각 요소가 존재하는지 확인한다음 호출할 수 있다.
```lua
E = {}     -- can be reused in other similar expressions
zip = (((company or E).director or E).address or E).zipcode
```

## The Table Library
table 라이브러리는 몇가지 유용한 함수들을 제공한다.  
`table.insert` 함수는 sequence의 원하는 위치에 요소를 삽입하고, 위치 다음에 있는 요소들은 한칸씩 뒤로 미룬다.  
`table.insert` 함수를 인자 없이 호출하면 sequence의 가장 뒤에 요소를 삽입한다.  
```lua
t = {}
for line in io.lines() do
  table.insert(t, line)
end
  
print(#t) --> (number of lines read)
```

`tables.remove` 함수는 sequence의 원하는 위치의 요소를 제거하고, 위치 다음에 있는 요소들을 앞으로 당긴다.  
`tables.remove` 함수를 인자 없이 호출하면 sequence의 마지막 요소를 제거한다.  

# Chapter 6. Functions

Lua 프로그램은 Lua로 작성된 함수, C로 작성된 함수(또는 host 어플리케이션이 사용하는 다른 모든 언어로 작성된 함수)를 사용할 수 있다.  
일반적으로는 성능 때문에, 그리고 Lua에서 직접적으로 사용하기 힘든 기능들(OS의 기능 등)을 사용하기 위해 C 함수를 사용한다.  
예를들면 표준 Lua 라이브러리는 C로 작성되었다.  
하지만 함수를 호출할 때는 Lua로 작성된 함수와 C로 작성된 함수에 차이가 없다.  

Lua로 작성된 함수는 이런 보편적인 문법을 가진다:
```lua
-- add the elements of sequence 'a'
function add (a)
  local sum = 0
  for i = 1, #a do
    sum = sum + a[i]
  end
  return sum
end
```

Lua 함수를 호출할 때는 인자수를 정확하게 맞추지 않아도 되는데, 이때의 동작은 다음과 같다:
```lua
-- 인자를 2개 받아서 출력하는 함수
function f (a, b) 
  print(a, b) 
end

f()        --> nil nil
f(3)       --> 3 nil
f(3, 4)    --> 3 4
f(3, 4, 5) --> 3 4 (5 is discarded)
```
초기화되지 않은 함수 파라미터는 함수 내에서 nil 값이 되고, 파라미터 수보다 더 많이 입력한 인자는 버려진다.  
이런 동작을 충분히 이해하고 있지 않으면 예상치 못한 오류가 발생할 수 있기 때문에 주의해야 한다.  

만약 함수내에서 초기화되지 않은 파라미터에 초기값을 지정하고 싶다면 or 연산자를 사용하면 된다.  
```lua
function incCount (n)
  n = n or 1    -- n이 초기화되지 않았으면 초기값을 1로 지정한다.
  globalCounter = globalCounter + n
end
```

## Multiple Results
Lua 함수는 여러개의 값을 리턴할 수 있다.  
예를들면 `string.find` 함수는 문자열에서 패턴이 시작하는 인덱스와 끝나는 인덱스 2개값을 리턴한다.  
```lua
s, e = string.find("hello Lua users", "Lua")
print(s, e)   --> 7 9
```

여러개의 리턴값을 가지려면 return 키워드 뒤에 여러개의 값을 나열한다.  
```lua
function maximum (a)
  local mi = 1          -- index of the maximum value
  local m = a[mi]       -- maximum value
  for i = 1, #a do
    if a[i] > m then
    mi = i; m = a[i]
    end
  end
  return m, mi          -- return the maximum and its index
end

print(maximum({8,10,23,12,5})) --> 23 3
```

Lua 함수는 함수가 호출되는 상황에 따라 리턴값의 개수를 조절한다.  
```lua
function foo0 () end                  -- returns no results
function foo1 () return "a" end       -- returns 1 result
function foo2 () return "a", "b" end  -- returns 2 results

-- 이렇게 단지 statement로 함수를 호출하면 리턴값은 모두 버려진다.
foo()    -- 리턴값 버려짐
foo1()   -- 리턴값 버려짐
foo2()   -- 리턴값 버려짐

-- 함수 결과를 변수에 저장할 경우 변수 개수만큼 리턴값이 생성된다.
x, y = foo2()         -- x="a", y="b"
x = foo2()            -- x="a", "b" is discarded
x, y, z = foo2()      -- x=10, y="a", z=nil

-- 만약 함수 호출 뒤에 다른 expression이 있다면, 함수의 리턴값은 단 1개만 생성된다.
x, y = foo2()              -- x="a", y="b"
x, y = foo2(), 20          -- x="a", y=20        (foo2() 호출 뒤에 다른 expression이 있어서 "b"는 버려졌다)
x, y, z = 10, foo2(), 20   -- x=10, y="a", z=20  (foo2() 호출 뒤에 다른 expression이 있어서 "b"는 버려졌다)

print(foo2())              --> a b (foo2() 호출 뒤에 다른 expression이 없기 때문에 리턴결과가 모두 사용되었다)
print(foo2(), 1)           --> a 1 (foo2() 호출 뒤에 다른 expression이 있어서 "b"는 버려졌다)
```

함수의 리턴값을 table의 생성자로 사용할 수 있다.
```lua
t = {foo0()}     -- t = {} (an empty table)
t = {foo1()}     -- t = {"a"}
t = {foo2()}     -- t = {"a", "b"}
```
이 때도 예외없이 함수 호출 뒤에 다른 expression이 있으면 함수의 리턴값은 단 1개만 생성된다.
```lua
t = {foo0(), foo2(), 4} -- t[1] = nil, t[2] = "a", t[3] = 4
```

함수 호출이 무조건 1개 값만 리턴하도록 하려면 호출을 ()로 감싸면 된다.
```lua
print((foo0()))    --> nil
print((foo1()))    --> a
print((foo2()))    --> a
```

`return` 문을 작성할 때 리턴값을 괄호로 감쌀때의 효과를 주의하자.  
```lua
return (f(x))   -- f 함수가 몇개의 값을 리턴하던간에 반드시 1개의 값만 리턴된다.
```

## Variadic Functions
Lua에서는 가변개수 파라미터를 가지는 함수를 `variadic` 함수라고 부른다.  
예를들면 print 함수가 variadic 함수이다.  
함수를 가변개수 파라미터로 선언하려면 ... 키워드를 쓰면 된다.
```lua
function add (...)  -- 가변개수로 인자를 받음
  local s = 0
  for _, v in ipairs{...} do  -- 가변개수 인자를 순회함
    s = s + v
  end
  return s
end

print(add(3, 4, 10, 25, 12)) --> 54
```
함수 내에서 가변개수 인자값을 지역변수에 저장하려면 이렇게 하면 된다.
```lua
local a, b = ...    -- 가변개수 인자 중 첫번째, 두번째 값을 a, b에 저장한다.
```

가변개수 인자를 순회하려면 `ipairs{...}` 를 사용하면 된다.
```lua
function var(...)
  for _, v in ipairs{...} do
    print(v)
  end
end

var(1,2,3)
-- 1
-- 2
-- 3
```
그런데 `ipairs{...}` 에는 한가지 문제점이 있는데, 인자 중간에 nil이 있다면 그곳부터는 더이상 순회할 수 없다는 것이다.
```lua
var(1,2,nil,3)
-- 1
-- 2 (다음값 nil은 순회되지 않음)
```

이런 문제를 해결하려면 `table.pack` 함수를 사용하면 된다.
`table.pack` 함수는 `...` 안의 모든 값들을 사용하여 table을 만들어준다.  
그리고 `table.pack` 함수로 만든 table에는 `"n"` 필드가 있는데, 여기에는 인자의 수가 들어있다.  
```lua
function var2(...)
  local t = table.pack(...)
  for i=1, t.n do
    print(t[i])
  end
end

var2(1,2,nil,3)
-- 1
-- 2
-- nil
-- 3
```

## The function table.unpack
`table.unpack` 함수는 list를 인자로 받아서 list의 모든 요소들을 분리하여 리턴해준다.  
```lua
print(table.unpack{10,20,30})   --> 10 20 30
a,b = table.unpack{10,20,30}    -- a=10, b=20, 30 is discarded
```
반대로 `table.pack`은 인자들을 1개의 list로 변경해준다.  

`table.unpack`을 사용하면 임의의 함수를 임의의 타입을 가지는 임의의 개수의 인자로 호출할 수 있다.  
예를들자면
```lua
f = string.find     -- 임의의 원하는 함수
a = {"hello", "ll"} -- 임의의 타입을 가지는 임의의 개수의 인자
f(table.unpack(a))  -- 함수 호출
```

`table.unpack` 함수는 리스트에 값이 몇개 있는지를 알기위해 길이 연산자를 사용한다.  
그래서 리스트가 올바른 sequence 일때만 사용할 수 있다.  

`table.unpack` 함수에는 리스트의 몇번째 값부터 몇번째 값까지를 unpack 할지를 결정할수있는 파라미터도 있다.  
```lua
print(table.unpack({"Sun", "Mon", "Tue", "Wed"}, 2, 3))  -- 리스트의 2번째 부터 3번째 값을 unpack 한다.
 --> Mon Tue
```

## Proper Tail Calls
Lua 함수의 흥미로운점은 Lua는 tail-call elimination을 사용한다는 것이다.  
tail call은 어떤 함수의 호출이 다른 함수가 마지막으로 수행할 명령어일때 발생한다.  
예를들면 아래에서 g 함수의 호출은 tail call 이다.
```lua
function f(x) 
  x = x + 1; 
  return g(x) -- g(x) 는 tail call 이다.
end
```
함수 f는 g를 호출한 다음에는 아무것도 할 일이 없다.  
이런 경우에는 프로그램은 g 함수가 종료된 다음 f 함수의 위치로 돌아갈 필요가 없다.  
그래서 g 함수가 호출된 다음에는 f 함수의 컨텍스트(stack 메모리공간 등)를 유지할 필요가 없다.  
g 함수가 종료되면, 프로그램의 흐름은 f 함수를 호출한 위치로 바로 이동된다.  
그래서 tail call이 발생했을 때는 stack 메모리 공간을 추가로 사용하지 않는다.  
이러한 구현방식을 tail-call elimination 이라고 한다.  

tail-call elimination을 사용하기 때문에 함수가 재귀호출을 하더라도 stack overflow가 발생하지 않는다.  
```lua
function foo(n)
  if n > 0 then 
    return foo(n - 1) 
  end
end

foo(100000000) -- stack overflow가 발생하지 않는다.
```

그런데 tail call 인지 아닌지를 판단하기는 약간 까다롭다.  
아래 코드에서 g의 호출은 tail call이 아니다.
```lua
function f(x) 
  g(x) 
end
```
왜냐하면 f가 g를 호출한 다음에 f는 g의 리턴값을 버려야 하기 때문이다.  
비슷하게 아래 예시도 tail call이 아니다.
```lua
return g(x) + 1   -- must do the addition
return x or g(x)  -- must adjust to 1 result
return (g(x))     -- must adjust to 1 result
```

Lua에서는 오직 `return func(args)` 형태만 tail call 이다.
