# Chapter 1. Getting Started

## Chunks
Lua가 실행하는 각각의 코드조각을 chunk라고 한다.  
chunk는 파일이나 1줄의 명령어일 수 있다. 예를들면 print("Hello Workd") 같은것 말이다.  
chunk의 크기는 아주 클 수도 있는데, Lua는 data-description 언어로도 사용되기 때문에 수 메가바이트 크기의 chunk도 있다.  

## 주석
주석은 -- 로 시작한다.  
여러줄의 주석은 --[[ 로 시작하고 --]] 로 끝난다.  
```lua
-- 주석

--[[
여러줄의
주석
--]]
```

## 전역변수
전역변수는 선언이 필요없다. 단지 사용하기만 하면 된다.  
```lua
b     -- 선언하지 않고 사용한 b는 초기값은 nil 이다.
b = 10
b     -- b는 10이다.
```

## 타입과 값
Lua는 dynamically-typed 언어이다. 그래서 변수의 타입을 선언할 필요가 없다.  
Lua에는 8개의 기본적인 type이 있다.  
nil, Boolean, number, string, userdata, function, thread, table 이다.  

type 함수는 값의 타입을 알려준다.  
```lua
type(nil)           --> nil
type(true)          --> boolean
type(10.4 * 3)      --> number
type("Hello world") --> string
type(io.stdin)      --> userdata
type(print)         --> function
type(type)          --> function
type({})            --> table
type(type(X))       --> string
```
userdata 타입은 임의의 C 데이터이다.  
userdata 타입에 대해서는 assignment와 equality test를 빼고는 정해진 연산이 없다.  
userdata는 Lua가 임베딩된 어플리케이션이나 C 라이브러리가 사용하는 임의의 타입을 표현하기위해 사용된다.  
예를들면 표준 I/O 라이브러리는 열린 파일을 나타내기 위해 userdata를 사용한다.  

변수는 정의된 타입이 없다. 모든 변수는 모든 타입의 값을 저장할 수 있다.
```lua
type(a)           --> nil ('a' is not initialized)
a = 10
type(a)           --> number
a = "a string!!"
type(a)           --> string
a = nil
type(a)           --> nil
```

## Nil
nil은 값이 없음을 나타내는데 사용된다.  
초기화되지 않은 처음 사용하는 변수는 nil 값이 입력되어 있다.  
만약 전역변수는 제거하고 싶다면 전역변수에 nil을 할당하면 된다.
```lua
a = nil    -- a 변수는 제거된다.
```

## Booleans
`true`, `false`는 Booleans 타입이다.  
Lua의 조건 검사는 `false` 와 `nil` 을 거짓으로 취급한다.  
나머지 모든 값은 참으로 취급한다.  
예를들면 0 과 빈 문자열도 참으로 취급한다.  

Lua는 `and`, `or`, `not` 조건연산자를 제공한다.  
`and` 연산자의 결과는, and의 첫번째 피연산자가 false라면 첫번째 피연산자, 그리고 첫번째 피연산자가 true라면 두번째 피연산자이다.  
`or` 연산자의 결과는, or의 첫번째 피연산자가 true라면 첫번째 피연산자, 그리고 첫번째 피연산자가 false라면 두번째 피연산자이다.  
```lua
4 and 5         --> 첫번째 피연산자가 참   --> 5
nil and 13      --> 첫번째 피연산자가 거짓 --> nil
false and 13    --> 첫번째 피연산자가 거짓 --> false
0 or 5          --> 첫번째 피연산자가 참   --> 0
false or "hi"   --> 첫번째 피연산자가 거짓 --> "hi"
nil or false    --> 첫번째 피연산자가 거짓 --> false
```

`and` 와 `or` 연산자 모두 short-circuit evaluation을 사용하기 때문에, 두번째 피연산자는 오직 첫번째 피연산자가 부정 일때만 연산된다.  

Lua 에는 자주 사용되는 구문이 있다.  
다음은 x가 초기화되지 않았을 때 x에 v값을 할당한다.  
```lua
if not x then x = v end
```
(그런데 x에 false가 입력되지 않았어야 할 것이다.)

또다른 유용한 구문은 다음과 같다.  
```lua
a and b or c
```
이 구문은 C의 `a ? b : c` 와 동일하다.  

`not` 연산자의 결과값은 언제나 Boolean 타입이다.  
```lua
not nil       --> true
not false     --> true
not 0         --> false
not not 1     --> true
not not nil   --> false
```

# Chapter 3. Numbers

Lua 버전 5.2 까지는 모든 숫자가 double-precision floating-point 형태였다.  
버전 5.3 부터는 숫자 표현에 64-bit integer가 추가되었다.  

## Numerals
숫자는 여러 형태로 사용할 수 있다.
```lua
4         --> 4
0.4       --> 0.4
4.57e-3   --> 0.00457
0.3e12    --> 300000000000.0
5E+20     --> 5e+20
```

integer와 float 값은 모두 `number` 타입이다.
```lua
type(3)     --> number
type(3.5)   --> number
type(3.0)   --> number
```

Lua 에서는 같은 값의 integer와 float은 동일하다.  
```lua
1 == 1.0      --> true
-3 == -3.0    --> true
0.2e3 == 200  --> true
```

integer와 float을 구분해야 한다면 `math.type` 함수를 사용한다.  
```lua
math.type(3)     --> integer
math.type(3.0)   --> float
```

16진수는 맨 앞에 0x 를 붙이면 된다.
```lua
0xff      --> 255
0x1A3     --> 419
0x0.2     --> 0.125
0x1p-1    --> 0.5
0xa.bp2   --> 42.75
```

## The Mathematical Library
Lua는 표준 `math` 라이브러리를 제공한다.  
이 라이브러리에는 삼각함수(sin, cos, tan, asin 등), 로그함수, 반올림 함수, max, min, 난수발생 함수 등이 있다.  
그리고 pi, huge(=infinite) 등의 상수값도 제공한다.  
```lua
ath.sin(math.pi / 2)      --> 1.0
math.max(10.4, 7, -3, 20) --> 20
math.huge                 --> inf
```

## Random-number generator
`math.random` 함수는 의사 난수를 생성한다.  
```lua
math.randomseed(os.time())   -- 시드 설정
math.random()                -- 0 이상 1 미만의 float 난수 생성
math.random(6)               -- 1 이상 6 이하의 integer 난수 생성
math.random(0,5)             -- 0 이상 5 이하의 integer 난수 생성
```

## Conversions
숫자를 float으로 변환하려면 0.0을 더하면 된다.  
```lua
-3 + 0.0                  --> -3.0
0x7fffffffffffffff + 0.0  --> 9.2233720368548e+18
```

숫자를 integer로 변환하려면 숫자에 rounding 함수를 사용한다.(math.ceil, math.floor, math.modf),  
```lua
math.ceil(3.3)    --> 4
math.ceil(-3.3)   --> -3
math.modf(3.3)    --> 3 0.3
math.modf(-3.3)   --> -3 -0.3
```
또는 math.tointeger 함수를 사용한다. math.tointeger 함수는 값에 소수 부분이 없어야 한다.  
```lua
math.tointeger(-258.0)      --> -258
math.tointeger(2^30)        --> 1073741824
math.tointeger(5.01)        --> nil (not an integral value)
math.tointeger(2^64)        --> nil (out of range)
```

# Chapter 5. Tables

Table은 Lua의 핵심 데이터 구조이다.  
Lua에서는 array, set, record, 그리고 다른 많은 데이터 형태들에 table을 사용한다.  
또한 package와 object를 표현하는데도 table을 사용한다.
예를들면 `math.sin` 함수를 생각해보면, math 라이브러리의 sin 함수라고 생각하겠지만, Lua에서는 math table에서 Key가 "sin"인 value이다.  

Lua의 table은 본질적으로 array이다. table은 숫자, 문자열 뿐만 아니라 Lua의 모든 값(nil 제외)을 index로 사용할 수 있는 array 이다.  

Lua의 table은 value도, variable도 아니다. table은 object이다.  

빈 table은 {} 표현으로 생성할 수 있다.
```lua
a = {}                -- create a table and assign its reference
k = "x"
a[k] = 10             -- new entry, with key="x" and value=10
a[20] = "great"       -- new entry, with key=20 and value="great"
a["x"]                --> 10
k = 20
a[k]                  --> "great"
a["x"] = a["x"] + 1   -- increments entry "x"
a["x"]                --> 11
```

table은 언제나 익명 객체이다. table 객체 자체와 table을 가르키는 변수명 사이에는 어떠한 고정된 관계도 없다.
```lua
a = {}
a["x"] = 10
b = a         -- 'b' refers to the same table as 'a'
b["x"]        --> 10
b["x"] = 20
a["x"]        --> 20
a = nil       -- only 'b' still refers to the table
b = nil       -- no references left to the table
```

## Table Indices
전역변수와 마찬가지로, table의 초기화되지 않은 요소는 초기값이 nil 이다.  
또한 요소에 nil을 할당하여 필드를 삭제할 수 있다.
```lua
a["y"]     --> nil
```

table의 요소에는 인덱스 말고 필드명 표현으로도 접근할 수 있다.  
예를들면 a.name 은 a["name"] 과 동일하다.  
```lua
a = {}         -- empty table
a.x = 10       -- same as a["x"] = 10
a.x --> 10     -- same as a["x"]
a.y --> nil    -- same as a["y"]
```

integer와 float 타입 index는 다음과 같다.
```lua
a = {}
a[2.0] = 10
a[2.1] = 20

a[2]      --> 10
a[2.1]    --> 20
```
2와 2.0은 동일한 key인데, table의 key가 숫자일 경우에는 만약 숫자값을 정수로 변환할 수 있다면 자동으로 변환되기 때문이다.  
정수로 변환될 수 없는 숫자값은 변환없이 그대로 사용된다.  

## Table Constructors
table은 다양한 방식으로 초기화할 수 있다.

{} 는 빈 table이다.

아래와 같이 초기화 한다면
```lua
days = {"Sunday", "Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday"}

print(days[4]) --> Wednesday
```
days[1] 은 Sunday, days[7] 은 Saturday 이다.  
이렇게 초기화할 경우 인덱스의 가장 처음 값이 0이 아니라 1인 것에 주의하자.  

또한 record-like table 형태의 초기화 방식도 제공한다.
```lua
a = {x = 10, y = 20}
```
이것은 아래와 동일하다.
```lua
a = {}
a.x = 10
a.y = 20
```

table 안에 table이 있는 형태의 초기화도 가능하다.
```lua
polyline = { color="blue", 
             thickness=2, 
             npoints=4,
             {x=0, y=0},      -- polyline[1]
             {x=-10, y=0},    -- polyline[2]
             {x=-10, y=1},    -- polyline[3]
             {x=0, y=1}       -- polyline[4]
           }
```

## Arrays, Lists, and Sequences
전통적인 array도 table로 만든다.  
table이 순서대로 나열된 정수값 key를 가지고 있다면(예: 1,2,3,4,5), 이것은 array 이다.  

array에 대해 생각해보자.  
array를 사용하려면 array의 길이를 알아야 한다.  
그리고 array는 시작 인덱스부터 마지막 인덱스까지 중간에 비어있는 곳이 없다.  
그런데 table은 이러한 규칙에는 완전히 위배된다.  
길이 라는것도 없고, 중간 인덱스가 없을수도 있기 때문이다.  

Lua에서는 table을 array처럼 사용하기 위해 sequence 라는것을 정의한다.  
sequence는 "1부터 양의 정수 n까지의 인덱스에 모두 nil이 아닌 value가 존재하는 리스트" 이다.  
특정 인덱스의 value가 nil라는 것은 존재하지 않는 인덱스이다.
그리고 Lua에서는 sequence의 길이를 알기 위해서 # 연산자를 제공한다.  
\# 연산자는 table 내의 sequence의 길이를 반환한다.  
아래 코드는 인덱스 1부터 마지막 인덱스(#a)까지 table내의 sequence를 순회하며 print 한다.  
```lua
-- print the lines, from 1 to #a
for i = 1, #a do
  print(a[i])
end
```

\# 연산자는 다음과 같이 sequence의 맨 뒤에 요소를 삽입할때도 사용된다.
```lua
a[#a + 1] = v   -- appends 'v' to the end of the sequence
```
만약 table에 양의정수 key를 가지는 요소가 없다면 해당 table의 길이(#)는 0이다.

## Table Traversal
table을 key-value 형태로 순회하려면 `pairs` iterator를 사용한다.
```lua
t = {10, print, x = 12, k = "hi"}
for k, v in pairs(t) do
  print(k, v)
    --> 1 10
    --> k hi
    --> 2 function: 0x420610
    --> x 12
end
```
table의 요소들이 순회되는 순서는 알 수 없다. table의 요소들은 정렬되어 있지 않기 때문이다.  

table을 list로 순회하려면 `ipairs` iterator를 사용한다.
```lua
t = {10, print, 12, "hi"}
for k, v in ipairs(t) do
  print(k, v)
    --> 1 10
    --> 2 function: 0x420610
    --> 3 12
    --> 4 hi
end
```
이 경우에는 table의 요소를 순서대로 순회한다.

table을 sequence로 순회하려면 정수 인덱스를 사용한다.
```lua
t = {10, print, 12, "hi"}
for k = 1, #t do
  print(k, t[k])
    --> 1 10
    --> 2 function: 0x420610
    --> 3 12
    --> 4 h
end
```

## Safe Navigation
라이브러리의 함수가 존재하는지 확인한 다음 호출하려면 어떻게 해야할까?  
아래처럼 zipcode 함수를 호출하려고 하는데, 먼저 존재하는지 확인하고 호출하려고 한다.
```lua
zip = company.director.address.zipcode
```

이 때는 or 연산자로 각 요소가 존재하는지 확인한다음 호출할 수 있다.
```lua
E = {}     -- can be reused in other similar expressions
zip = (((company or E).director or E).address or E).zipcode
```

## The Table Library
table 라이브러리는 몇가지 유용한 함수들을 제공한다.  
`table.insert` 함수는 sequence의 원하는 위치에 요소를 삽입하고, 위치 다음에 있는 요소들은 한칸씩 뒤로 미룬다.  
`table.insert` 함수를 인자 없이 호출하면 sequence의 가장 뒤에 요소를 삽입한다.  
```lua
t = {}
for line in io.lines() do
  table.insert(t, line)
end
  
print(#t) --> (number of lines read)
```

`tables.remove` 함수는 sequence의 원하는 위치의 요소를 제거하고, 위치 다음에 있는 요소들을 앞으로 당긴다.  
`tables.remove` 함수를 인자 없이 호출하면 sequence의 마지막 요소를 제거한다.  

# Chapter 6. Functions

Lua 프로그램은 Lua로 작성된 함수, C로 작성된 함수(또는 host 어플리케이션이 사용하는 다른 모든 언어로 작성된 함수)를 사용할 수 있다.  
일반적으로는 성능 때문에, 그리고 Lua에서 직접적으로 사용하기 힘든 기능들(OS의 기능 등)을 사용하기 위해 C 함수를 사용한다.  
예를들면 표준 Lua 라이브러리는 C로 작성되었다.  
하지만 함수를 호출할 때는 Lua로 작성된 함수와 C로 작성된 함수에 차이가 없다.  

Lua로 작성된 함수는 이런 보편적인 문법을 가진다:
```lua
-- add the elements of sequence 'a'
function add (a)
  local sum = 0
  for i = 1, #a do
    sum = sum + a[i]
  end
  return sum
end
```

Lua 함수를 호출할 때는 인자수를 정확하게 맞추지 않아도 되는데, 이때의 동작은 다음과 같다:
```lua
-- 인자를 2개 받아서 출력하는 함수
function f (a, b) 
  print(a, b) 
end

f()        --> nil nil
f(3)       --> 3 nil
f(3, 4)    --> 3 4
f(3, 4, 5) --> 3 4 (5 is discarded)
```
초기화되지 않은 함수 파라미터는 함수 내에서 nil 값이 되고, 파라미터 수보다 더 많이 입력한 인자는 버려진다.  
이런 동작을 충분히 이해하고 있지 않으면 예상치 못한 오류가 발생할 수 있기 때문에 주의해야 한다.  

만약 함수내에서 초기화되지 않은 파라미터에 초기값을 지정하고 싶다면 or 연산자를 사용하면 된다.  
```lua
function incCount (n)
  n = n or 1    -- n이 초기화되지 않았으면 초기값을 1로 지정한다.
  globalCounter = globalCounter + n
end
```

## Multiple Results
Lua 함수는 여러개의 값을 리턴할 수 있다.  
예를들면 `string.find` 함수는 문자열에서 패턴이 시작하는 인덱스와 끝나는 인덱스 2개값을 리턴한다.  
```lua
s, e = string.find("hello Lua users", "Lua")
print(s, e)   --> 7 9
```

여러개의 리턴값을 가지려면 return 키워드 뒤에 여러개의 값을 나열한다.  
```lua
function maximum (a)
  local mi = 1          -- index of the maximum value
  local m = a[mi]       -- maximum value
  for i = 1, #a do
    if a[i] > m then
    mi = i; m = a[i]
    end
  end
  return m, mi          -- return the maximum and its index
end

print(maximum({8,10,23,12,5})) --> 23 3
```

Lua 함수는 함수가 호출되는 상황에 따라 리턴값의 개수를 조절한다.  
```lua
function foo0 () end                  -- returns no results
function foo1 () return "a" end       -- returns 1 result
function foo2 () return "a", "b" end  -- returns 2 results

-- 이렇게 단지 statement로 함수를 호출하면 리턴값은 모두 버려진다.
foo()    -- 리턴값 버려짐
foo1()   -- 리턴값 버려짐
foo2()   -- 리턴값 버려짐

-- 함수 결과를 변수에 저장할 경우 변수 개수만큼 리턴값이 생성된다.
x, y = foo2()         -- x="a", y="b"
x = foo2()            -- x="a", "b" is discarded
x, y, z = foo2()      -- x=10, y="a", z=nil

-- 만약 함수 호출 뒤에 다른 expression이 있다면, 함수의 리턴값은 단 1개만 생성된다.
x, y = foo2()              -- x="a", y="b"
x, y = foo2(), 20          -- x="a", y=20        (foo2() 호출 뒤에 다른 expression이 있어서 "b"는 버려졌다)
x, y, z = 10, foo2(), 20   -- x=10, y="a", z=20  (foo2() 호출 뒤에 다른 expression이 있어서 "b"는 버려졌다)

print(foo2())              --> a b (foo2() 호출 뒤에 다른 expression이 없기 때문에 리턴결과가 모두 사용되었다)
print(foo2(), 1)           --> a 1 (foo2() 호출 뒤에 다른 expression이 있어서 "b"는 버려졌다)
```

함수의 리턴값을 table의 생성자로 사용할 수 있다.
```lua
t = {foo0()}     -- t = {} (an empty table)
t = {foo1()}     -- t = {"a"}
t = {foo2()}     -- t = {"a", "b"}
```
이 때도 예외없이 함수 호출 뒤에 다른 expression이 있으면 함수의 리턴값은 단 1개만 생성된다.
```lua
t = {foo0(), foo2(), 4} -- t[1] = nil, t[2] = "a", t[3] = 4
```

함수 호출이 무조건 1개 값만 리턴하도록 하려면 호출을 ()로 감싸면 된다.
```lua
print((foo0()))    --> nil
print((foo1()))    --> a
print((foo2()))    --> a
```

`return` 문을 작성할 때 리턴값을 괄호로 감쌀때의 효과를 주의하자.  
```lua
return (f(x))   -- f 함수가 몇개의 값을 리턴하던간에 반드시 1개의 값만 리턴된다.
```

## Variadic Functions
Lua에서는 가변개수 파라미터를 가지는 함수를 `variadic` 함수라고 부른다.  
예를들면 print 함수가 variadic 함수이다.  
함수를 가변개수 파라미터로 선언하려면 ... 키워드를 쓰면 된다.
```lua
function add (...)  -- 가변개수로 인자를 받음
  local s = 0
  for _, v in ipairs{...} do  -- 가변개수 인자를 순회함
    s = s + v
  end
  return s
end

print(add(3, 4, 10, 25, 12)) --> 54
```
함수 내에서 가변개수 인자값을 지역변수에 저장하려면 이렇게 하면 된다.
```lua
local a, b = ...    -- 가변개수 인자 중 첫번째, 두번째 값을 a, b에 저장한다.
```

가변개수 인자를 순회하려면 `ipairs{...}` 를 사용하면 된다.
```lua
function var(...)
  for _, v in ipairs{...} do
    print(v)
  end
end

var(1,2,3)
-- 1
-- 2
-- 3
```
그런데 `ipairs{...}` 에는 한가지 문제점이 있는데, 인자 중간에 nil이 있다면 그곳부터는 더이상 순회할 수 없다는 것이다.
```lua
var(1,2,nil,3)
-- 1
-- 2 (다음값 nil은 순회되지 않음)
```

이런 문제를 해결하려면 `table.pack` 함수를 사용하면 된다.
`table.pack` 함수는 `...` 안의 모든 값들을 사용하여 table을 만들어준다.  
그리고 `table.pack` 함수로 만든 table에는 `"n"` 필드가 있는데, 여기에는 인자의 수가 들어있다.  
```lua
function var2(...)
  local t = table.pack(...)
  for i=1, t.n do
    print(t[i])
  end
end

var2(1,2,nil,3)
-- 1
-- 2
-- nil
-- 3
```

## The function table.unpack
`table.unpack` 함수는 list를 인자로 받아서 list의 모든 요소들을 분리하여 리턴해준다.  
```lua
print(table.unpack{10,20,30})   --> 10 20 30
a,b = table.unpack{10,20,30}    -- a=10, b=20, 30 is discarded
```
반대로 `table.pack`은 인자들을 1개의 list로 변경해준다.  

`table.unpack`을 사용하면 임의의 함수를 임의의 타입을 가지는 임의의 개수의 인자로 호출할 수 있다.  
예를들자면
```lua
f = string.find     -- 임의의 원하는 함수
a = {"hello", "ll"} -- 임의의 타입을 가지는 임의의 개수의 인자
f(table.unpack(a))  -- 함수 호출
```

`table.unpack` 함수는 리스트에 값이 몇개 있는지를 알기위해 길이 연산자를 사용한다.  
그래서 리스트가 올바른 sequence 일때만 사용할 수 있다.  

`table.unpack` 함수에는 리스트의 몇번째 값부터 몇번째 값까지를 unpack 할지를 결정할수있는 파라미터도 있다.  
```lua
print(table.unpack({"Sun", "Mon", "Tue", "Wed"}, 2, 3))  -- 리스트의 2번째 부터 3번째 값을 unpack 한다.
 --> Mon Tue
```

## Proper Tail Calls
Lua 함수의 흥미로운점은 Lua는 tail-call elimination을 사용한다는 것이다.  
tail call은 어떤 함수의 호출이 다른 함수가 마지막으로 수행할 명령어일때 발생한다.  
예를들면 아래에서 g 함수의 호출은 tail call 이다.
```lua
function f(x) 
  x = x + 1; 
  return g(x) -- g(x) 는 tail call 이다.
end
```
함수 f는 g를 호출한 다음에는 아무것도 할 일이 없다.  
이런 경우에는 프로그램은 g 함수가 종료된 다음 f 함수의 위치로 돌아갈 필요가 없다.  
그래서 g 함수가 호출된 다음에는 f 함수의 컨텍스트(stack 메모리공간 등)를 유지할 필요가 없다.  
g 함수가 종료되면, 프로그램의 흐름은 f 함수를 호출한 위치로 바로 이동된다.  
그래서 tail call이 발생했을 때는 stack 메모리 공간을 추가로 사용하지 않는다.  
이러한 구현방식을 tail-call elimination 이라고 한다.  

tail-call elimination을 사용하기 때문에 함수가 재귀호출을 하더라도 stack overflow가 발생하지 않는다.  
```lua
function foo(n)
  if n > 0 then 
    return foo(n - 1) 
  end
end

foo(100000000) -- stack overflow가 발생하지 않는다.
```

그런데 tail call 인지 아닌지를 판단하기는 약간 까다롭다.  
아래 코드에서 g의 호출은 tail call이 아니다.
```lua
function f(x) 
  g(x) 
end
```
왜냐하면 f가 g를 호출한 다음에 f는 g의 리턴값을 버려야 하기 때문이다.  
비슷하게 아래 예시도 tail call이 아니다.
```lua
return g(x) + 1   -- must do the addition
return x or g(x)  -- must adjust to 1 result
return (g(x))     -- must adjust to 1 result
```

Lua에서는 오직 `return func(args)` 형태만 tail call 이다.

## Local Variables and Blocks
기본적으로 Lua의 변수는 전역변수이다.  
지역변수를 사용하려면 `local` 키워드로 선언해야 한다.  
지역변수는 지역변수가 선언된 블록 내에서 유효하다.  

```lua
x = 10         -- 전역변수 x
i = 1          -- 전역변수 i
if i < 20 then
  local x      -- 지역변수 x
  x = 20       -- 지역변수 x에 20 할당
  print(x + 2) --> 22 (지역변수 x)
else
  print(x)     --> 10 (전역변수 x)
end

print(x)       --> 10 (전역변수 x)
```
지역변수도 전역변수와 마찬가지로 초기값은 nil 이다.

지역변수는 블록 내에서 유효하다.  
그런데 Lua에는 다른 언어의 { } 와 같은 블록이 없다.  
다른 언어의 { } 와 같은 블록을 사용하고 싶다면 `do` 블록을 사용하면 된다.  
```lua
local x1, x2
do
  local a2 = 2*a                  -- a2 는 do 블록 내에서만 유효하다.
  local d = (b^2 - 4*a*c)^(1/2)   -- d 는 do 블록 내에서만 유효하다.
  x1 = (-b + d)/a2
  x2 = (-b - d)/a2
end                  -- scope of 'a2' and 'd' ends here
print(x1, x2)        -- 'x1' and 'x2' still in scop
```

지역변수를 쓸 수 있는 경우라면 가능한 전역변수 대신 지역변수를 쓰는것이 좋다.  

Lua의 `strict.lua` 모듈을 사용하면 전역변수에 대한 좀더 엄격한 검사가 추가된다.  
이 모듈을 사용하면 함수내에서 초기화되지 않은 전역변수를 사용하려고 하면 오류가 발생한다.  

## Control Structures
#### if then else
```lua
if op == "+" then
  r = a + b
elseif op == "-" then
  r = a - b
elseif op == "*" then
  r = a*b
elseif op == "/" then
  r = a/b
else
  error("invalid operation")
end
```
#### while
`while` 문은 조건이 true일 동안 반복한다.
```lua
local i = 1
while a[i] do
  print(a[i])
  i = i + 1
end
```
#### repeat
`repeat-until` 문은 조건이 true일 동안 반복한다. 블록을 무조건 한번 실행하고 조건을 검사한다.
```lua
-- computes the square root of 'x' using Newton-Raphson method
local sqr = x / 2
repeat
  sqr = (sqr + x/sqr) / 2
  local error = math.abs(sqr^2 - x)
until error < x/10000                -- local 'error' still visible here
```
#### Numerical for
`for` 문은 2가지 타입이 있다: `numerical for`, `generic for`
`numerical for` 문법은 다음과 같다:
```lua
for i = start, end, inc do
  something
end
```
start 부터 end 까지 순회하며, i 값은 inc 만큼씩 증가한다.  
inc를 생략하면 interval 값은 1이다.  
start, end, inc 값은 for 문이 시작하기 전에 1번만 계산된다.  
그리고 i 값은 for문 내에서만 존재하는 지역변수이다.  

#### Generic for
generic for는 iterator 함수의 리턴값을 모두 순회한다.  
generic for의 iterator는 `pairs`, `ipairs`, `io.lines` 등이 있다.  
generic for는 사용법이 약간 복잡하기 때문에 Chapter 18, Iterators and the Generic for 에서 소개하도록 하겠다.  

#### break, return, and goto
`goto`문은 loop 내에서 continue, redo를 구현하는데 사용할 수 있다.  
```lua
i = 1
while i < 10 do
  ::redo::            -- redo label
  i = i + 1
  if i % 2 == 0 then 
    goto continue     -- continue label로 이동
  end
  
  if i % 3 == 0 then
    goto redo         -- redo label로 이동
  end
  
  print(i)
 ::continue::          -- continue label
end
-- 출력결과:
-- 5
-- 7
```
`goto` 문에는 사용 조건이 있다.  
1. label은 변수의 scope 규칙을 따른다. 그래서 블록 밖에서 블록 내의 label로 jump할 수 없다. 왜냐하면 블록 내의 label은 블록 내에서만 존재하기 때문이다.
2. 함수의 밖으로 jump할 수 없다.

## Functions as First-Class Values
function은 "first-class Values"이다.  
이 말은 function은 숫자나 string 처럼 변수, table 등에에 저장될 수 있다는 말이다.  
그리고 함수의 인자로 function을 전달할 수도 있고, 함수가 function을 리턴할 수도 있다.  
```lua
a = {p = print}        -- 'a.p' refers to the 'print' function
a.p("Hello World")     --> Hello World
print = math.sin       -- 'print' now refers to the sine function
a.p(print(1))          --> 0.8414709848079
math.sin = a.p         -- 'sin' now refers to the print function
math.sin(10, 20)       --> 10 20

foo = function (x) return 2*x end  -- foo stores function
```
Lua 에서는 모든 함수는 익명이다. 다른 모든 value와 같이 이름을 가지지 않는다.  
우리가 print 같은 함수의 이름에 대해 이야기하는 것은, 사실은 함수를 저장하고 있는 print 변수명을 이야기하는 것이다.  

다음은 함수를 인자로 받는 함수인 table.sort 함수의 예시이다.
```lua
network = {
  {name = "grauna", IP = "210.26.30.34"},
  {name = "arraial", IP = "210.26.30.23"},
  {name = "lua", IP = "210.26.23.12"},
  {name = "derain", IP = "210.26.23.20"},
}

-- sort 함수의 두번째 인자로 정렬순서를 결정하는 함수를 전달하였다.
table.sort(network, function (a,b) return (a.name > b.name) end)
```

## Non-Global Functions
함수는 table이나 지역변수에도 저장될 수 있다.  
대부분의 Lua 라이브러리는 이 방식으로 만들어진다(io.read, math.sin 등).  
```lua
Lib = {}
Lib.foo = function (x,y) return x + y end
Lib.goo = function (x,y) return x - y end

 print(Lib.foo(2, 3), Lib.goo(2, 3))   --> 5 -1
```
또는 이런식으로도 함수를 선언할 수 있다.
```lua
Lib = {}
function Lib.foo (x,y) return x + y end
function Lib.goo (x,y) return x - y end
```
이 방식의 함수 선언은 객체지향 프로그래밍에서 유용하게 사용된다.  

지역변수에 저장하거나 local 키워드로 선언한 함수를 local function 이라고 한다.  
local function은 블록 scope 내에서만 존재한다.  
```
local function f (params)
  body
end
```
local function은 라이브러리를 개발할 때 유용한데, 라이브러리 내에서만 사용되는 함수를 local function으로 선언하면 라이브러리 밖에서는 볼 수 없게 된다.  

local function을 잘못 사용하면 아래와 같이 재귀 문제가 발생할 수 있다.  
```lua
local fact = function (n)
  if n == 0 then return 1
  else return n*fact(n-1) -- buggy
  end
end
```
fact 함수 내에서 fact 함수를 호출하고 있는데, 함수 내에서는 fact 함수가 아직 없기 때문에 오류가 발생하거나 전역 fact 함수가 사용된다.  
이런 문제를 피하려면 지역변수를 전방선언 하면 된다.  
```lua
local fact
fact = function (n)
  if n == 0 then return 1
  else return n*fact(n-1)
  end
end
```
이렇게 하면 fact 함수 내에서 fact 함수를 호출할 때, fact 변수에 함수가 저장되어 있을 것이기 때문에 문제없다.  

아래와 같은 간접적인 재귀함수가 발생할 수도 있다. 이 때에는 local function을 아래와 같이 선언하면 된다.
```lua
local f   -- local f 전방선언

-- local g 안에서는 local f를 호출한다.
local function g ()
  some code f() some code
end

-- local f 안에서는 local g를 호출한다.
-- 주의할 점은 여기에는 local 키워드를 붙이면 안된다는 것이다.
-- 왜냐하면 local function f() 로 선언하면 새로운 local f 변수에 함수가 저장되고, 전방선언한 local f에는 저장되지 않기 때문이다.
function f ()
  some code g() some code
end
```

## Lexical Scoping
어떤 함수A 안에서 선언된 함수B는 함수 A의 모든 지역변수에 접근권한이 있다.  
이러한 특성을 `lexical scoping` 이라고 한다.  

예시를 보도록 하자. 아래 sortbygrade 함수는 table.sort 함수에 비교함수를 전달하여 정렬한다.  
```lua
function sortbygrade (names, grades)
  table.sort(names, function (n1, n2)
                      return grades[n1] > grades[n2] -- compare the grades
                    end)
end
```
비교함수는 함수 내에서 grades 변수를 사용하고 있다.  
그런데 grades 변수는 sortbygrade 함수의 파라미터이기 때문에 sortbygrade 함수의 지역변수이다.  
비교함수는 sortybygrade 함수의 지역변수를 접근하여 사용하고 있다.  

다음 예시를 보도록 하자. 아래 newCounter 함수는 어떤 익명 함수를 생성하여 리턴한다.
```lua
function newCounter ()
  local count = 0
  return function () -- anonymous function
           count = count + 1
           return count
         end
end

c1 = newCounter()
print(c1()) --> 1
print(c1()) --> 2

c2 = newCounter()
print(c2()) --> 1
print(c1()) --> 3
print(c2()) --> 2
```
익명 함수는 newCounter 함수의 지역변수인 count에 접근한다.  
여기서 의문점은, 지역변수 count는 newCounter 함수가 종료되면 사라진다는 것이다.  
그런데 어떻게 익명함수는 계속해서 count에 접근할 수 있을까?  
심지어 출력결과를 보면 c1과 c2는 서로다른 함수를 가르키고 있고, c1과 c2 안의 count 변수는 각각 독립적으로 유지되는 것으로 보인다.  
이것은 함수가 `closure` 이기 때문에 가능하다.  

기술적으로 말하자면 Lua의 value는 function이 아니라 closure이다.  
function 자체는 closure의 프로토타입이다.  
closure는 () 연산자로 호출 가능한 객체로, 호출할 함수와 함수내에서 사용할 자신만의 변수를 가진다.  
newCounter 함수를 호출할 때마다 새로운 closure가 생성되고, closure는 호출할 함수(생성된 익명함수)와 자신만의 변수(count)를 가진다.  
그래서 count 변수에 접근할 수 있고 count의 값이 유지된다.  

다음은 함수의 closure 특성을 이용하여 코드에서 파일 접근제한 기능을 구현하는 예시이다.
```lua
do                                             -- do 블록으로 지역변수의 스코프를 정한다.
  local oldOpen = io.open                      -- 먼저 io.open 함수를 지역변수에 저장해둔다.
  local access_OK = function (filename, mode)  -- 파일이 접근 가능한지 체크하는 함수 선언
                      check access
                    end
  io.open = function (filename, mode)            -- io.open 함수를 재정의한다.
              if access_OK(filename, mode) then  -- 만약 파일이 접근 가능하다면
                return oldOpen(filename, mode)   -- 원래의 io.open 함수로 파일을 연다.
              else
                return nil, "access denied"      -- 파일이 접근 불가하다면 nil을 리턴한다.
              end
            end
end
```
이 코드를 실행하면 사용자가 io.open 함수를 사용할 때 접근제한 기능이 들어간 io.open 함수가 사용된다.  
그리고 원본 io.open 함수는 do 블록 내의 지역변수에 저장해뒀기 때문에 외부에서 접근할 수가 없게 된다.  
변경된 io.open 함수는 closure 특성을 이용하여 지역변수 oldOpen에 접근할 수 있기 때문에 원본 io.open 함수를 사용할 수 있다.

## Queues and Double-Ended Queues
Double-Ended Queue는 첫번째 요소의 index와 마지막 요소의 index를 저장하고있는 table로 구현할 수 있다.  

```lua
-- 빈 Double-Ended QUeue 생성하기
function listNew()
  return {first = 0, last = -1}  -- 첫번째 요소 인덱스와 마지막 요소 인덱스를 가진다.
end

-- Queue의 가장 앞에 요소 삽입하기
function pushFirst (list, value)
  local first = list.first - 1
  list.first = first
  list[first] = value
end

-- Queue의 가장 마지막에 요소 삽입하기
function pushLast (list, value)
  local last = list.last + 1
  list.last = last
  list[last] = value
end

-- Queue의 가장 앞 요소 pop하기
function popFirst (list)
  local first = list.first
  if first > list.last then error("list is empty") end
  local value = list[first]
  list[first] = nil                 -- to allow garbage collection
  list.first = first + 1
  return value
end

-- Queue의 가장 마지막 요소 pop하기
function popLast (list)
  local last = list.last
  if list.first > last then error("list is empty") end
  local value = list[last]
  list[last] = nil                -- to allow garbage collection
  list.last = last - 1
  return value
end

-- 테스트
list = listNew()
pushFirst(list, 3)
pushLast(list, 4)
pushFirst(list, 2)
pushFirst(list, 1)
pushLast(list, 5)
pushLast(list, 6)

print(popFirst(list))   -- 1
print(popFirst(list))   -- 2
print(popLast(list))    -- 6
print(popLast(list))    -- 5
print(popFirst(list))   -- 3
print(popLast(list))    -- 4
print(popLast(list))    -- error
```

```
local s = table.concat(t, "\n")   -- table의 요소들을 합쳐서 하나의 문자열로 만든다. 이 때 구분자는 "\n" 을 사용한다.
```

Chapter 16. Compilation, Execution, and Errors
Lua가 인터프리터 언어이기는 하지만 Lua는 소스코드를 실행하기 전에 항상 중간 형태로 컴파일한다(사실 대부분의 인터프리터 언어가 이렇게 한다).

하지만 `dofile`은 사실 보조 함수이다.

## Compilation
이전에 `dofile` 함수를 Lua에서 chunk를 실행하는 primitive operation으로 소개한 적이 있다.  
하지만 `dofile`은 사실 보조 함수이다.  
실제는 `loadfile` 함수가 핵심 역할을 한다.  
`loadfile`은 파일에서 chunk를 로드한다. 하지만 chunk를 실행하지는 않는다.  
대신 chunk를 컴파일한 다음 컴파일된 chunk를 함수의 형태로 리턴한다.  
그리고 `loadfile`은 `dofile`과는 달리 error를 발생시키지 않는다. 대신 error code를 리턴한다.  
`dofile` 함수의 내부는 이런식으로 되어있다.
```lua
function dofile (filename)
  local f = assert(loadfile(filename))
  return f()
end
```
`assert`는 `loadfile`이 실패했을 때 error를 발생시킨다.  

`loadfile`은 `dofile`보다 좀 더 유연한데, `loadfile`은 오류가 발생했을 때 nil과 오류메시지를 리턴한다. 그래서 사용자가 원하는대로 오류를 처리할 수 있다.  
그리고 `loadfile`의 리턴값을 f에 저장한 다음, 나중에 f를 원하는 만큼 호출할 수 있다.  
그래서 불필요한 컴파일 횟수를 줄일 수 있다.  

`load`함수는 `loadfile`과 비슷한데, 문자열을 통해 chunk를 입력받아 컴파일한 다음 함수의 형태로 리턴한다.  
```lua
f = load("i = i + 1")

i = 0
f(); print(i)   --> 1
f(); print(i)   --> 2
```
`load` 함수는 강력하지만, 속도가 느리고 잘못된 코드를 실행할 경우 알수없는 오류가 발생할 수 있기 때문에 웬만하면 사용하지 않는것이 좋다.  

`load`는 lexical scoping이 적용되지 않는다.  
```lua
i = 32
local i = 0

f = load("i = i + 1; print(i)")
g = function() i = i + 1; print(i) end

print(f())   --> 33 (global i 사용)
print(f())   --> 34 (global i 사용)
print(g())   --> 1 (캡처된 local i 사용)
print(g())   --> 2 (캡처된 local i 사용)
```

`load` 함수는 프로그램 외부에서 받은 코드를 동적으로 실행시킬때 주로 사용한다.  
예를들면 사용자로부터 입력받은 방정식으로 그래프를 그릴 때 사용할 수 있다.  

`load`와 `loadfile` 함수는 절대로 error를 발생시키지 않는다.  
오류상황일 경우, nil과 오류메시지를 리턴한다.
```lua
print(load("i i"))
  --> nil     [string "i i "]:1: syntax error near 'i'
```
그리고 `load`와 `loadfile` 함수는 단지 컴파일을 할 뿐, 절대 코드를 실행하지는 않는다.  
코드가 실행되는 순간은 이 함수로 리턴받은 익명함수를 호출할 때이다.  

만약 외부에서 전달받은 코드를 실행해야 한다면, 먼저 chunk를 컴파일 할 때 오류가 있는지 확인해야 한다.  
그리고 코드를 실제로 실행할 때는 시스템 환경이 망가지는것을 방지하기 위해 protected environment에서 실행해야 한다.  
이것은 Chapter 22, The Environment에서 다룰 예정이다.  

## Precompiled Code
Lua는 소스코드를 실행하기 전에 먼저 컴파일한다(precompile).  
Lua는 또한 소스코드를 precompiled 형태로 배포하는 기능도 제공한다.  

precompiled file(Lua 용어로 binary chunk)을 만드는 가장 간단한 방법은 `luac` 프로그램을 사용하는 것이다.  
예를들면 아래 명령어는 `prog.lua` 파일을 입력받아 이것의 precompiled file인 `prog.lc`를 생성한다.
```
$ luac -o prog.lc prog.lua
```
Lua 인터프리터는 `prog.lc` 파일을 일반적인 Lua 코드처럼 실행할 수 있다.  
```
$ lua prog.lc
```
또한 `loadfile`과 `load` 함수도 precompiled code를 인자로 받을 수 있다.  

Lua 코드로 최소 기능의 `luac`를 곧바로 만들 수 있다:
```lua
-- lua 소스코드 파일 로드
p = loadfile(arg[1])
-- precompiled file을 생성한다.
f = io.open(arg[2], "wb")
-- Lua 함수를 precompiled code 형태 문자열로 변환하고, 파일에 write한다.
f:write(string.dump(p))
-- 파일을 닫는다.
f:close()
```

precompiled code는 Lua 코드보다 크기가 항상 작은것은 아니지만, 더 빠르게 로드된다.  
그리고 precompiled code는 Lua 코드가 실수로 변경되는 일을 막아준다.  
다시 컴파일 하지 않는 이상은 바이너리는 변경되지 않을것이기 때문이다.  

## Errors
Lua에서는 `error` 함수로 error를 발생시킬 수 있다.
```lua
print "enter a number:"
n = io.read("n")
if not n then error("invalid input") end
```

`assert` 함수는 조건과 오류메시지를 입력받아서 조건이 맞지 않으면 error를 발생시킨다.
```lua
print "enter a number:"
n = assert(io.read("*n"), "invalid input")
```
`assert` 함수의 주의점은, `assert`는 일반적인 함수이기 때문에 함수를 호출하기 전 인자들을 모두 계산한다는 것이다.  
그래서 오류메시지를 문자열 붙이기로 만들 경우에는 `assert` 함수가 호출될 때마다 불필요한 문자열 붙이기가 발생할 수 있다.
```lua
n = io.read()
assert(tonumber(n), "invalid input: " .. n .. " is not a number") -- assert 함수가 호출될 때마다 문자열 붙이기로 문자열을 만듬
```

## Error Handling and Exceptions
대부분의 어플리케이션에서는 에러 핸들링을 Lua로 하지 않는다.  
Lua를 임베드한 프로그램이 에러를 핸들링한다.  
일반적으로 프로그램은 Lua에게 chunk 실행 요청만을 한다.  
만약 에러가 있다면 Lua는 에러 코드를 리턴할 것이고, 어플리케이션은 코드에 따라 적절한 조치를 취할 것이다.  

그런데 Lua 코드 내에서 에러 핸들링을 하고싶다면, `pcall`(protected call) 함수를 사용하여 코드를 감싸야 한다.  
다음은 `pcall` 함수로 오류 발생 가능성이 있는 코드를 실행하고 에러를 핸들링하는 방법이다.
```lua
local ok, msg = pcall(function ()
  some code
  if unexpected_condition then error() end
  some code
  print(a[i])    -- potential error: 'a' may not be a table
  some code
end)

if ok then     -- no errors while running protected code
  regular code
else           -- protected code raised an error: take appropriate action
  error-handling code
end
```
`pcall` 함수는 첫번째 인자로 받은 함수를 protected mode에서 호출하여 함수에서 발생하는 모든 에러를 catch한다.  
만약 에러가 없다면 `pcall` 함수는 true와 인자로 받은 함수의 리턴값을 리턴한다.  
만약 에러가 있다면 false와 에러 메시지를 리턴한다.

에러 핸들링 테스트1
```lua
local x = 0
local ok, msg = pcall(function ()
  x = x + 1  
  if x > 0 then error({msg="invalid x value", x=x}) end
end)

if ok then
  print("no error")
else
  print("error occurred. msg=" .. msg.msg .. ", x=" .. msg.x)
end
-- 출력결과: error occurred. msg=invalid x value, x=1
```

에러 핸들링 테스트2
```lua
local x = 0
local ok, msg = pcall(function ()
  x[1] = true
end)

if ok then
  print("no error")
else
  print("error occurred. msg=" .. msg)
end
-- 출력결과: error occurred. msg=attempt to index a number value (upvalue 'x')
```
