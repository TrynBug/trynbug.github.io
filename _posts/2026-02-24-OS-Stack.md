## Stack
Stack(User Stack)은 스레드가 실행되면서 사용할 지역변수, 함수파라미터, 함수 종료 후 돌아갈 주소 등을 저장하는 메모리 영역이다.  
모든 스레드는 자신만의 Stack을 가진다.  

Stack 영역 크기의 기본값은 1MB 이다.  
프로젝트 설정에서 기본값을 변경할 수 있지만 변경할 일은 잘 없다.  
스레드가 처음 생성되면 Stack 영역으로 사용할 1MB의 가상메모리를 reserve 한다.  
그런다음 처음 1Page(4KB)를 commit하여 실제로 사용할 수 있도록 한다.  
그리고 그다음 1Page(4KB)는 guard page로 지정해둔다.  

guard page는 가상메모리 영역에 PAGE_GUARD 속성을 지정한 page를 말하며, 읽기/쓰기 불가로 설정되어 있다.  
CPU가 이 영역에 접근하려고 시도하면 EXCEPTION_GUARD_PAGE 예외가 발생하고, OS는 예외를 받아서 Stack 메모리영역을 1Page 추가로 commit 한다.  

Stack 영역은 실제 사용하는 만큼만 commit하기 때문에 Stack 기본 크기가 1MB라고 해서 메모리를 낭비하는것이 아니다.  
Stack 영역은 commit한 크기가 늘어날 수는 있어도 줄어들 수는 없다.

## 커널 스택
커널 스택(Kernel Stack)은 스레드가 커널모드에서 작업해야할 때 사용하는 Stack 메모리영역이다.  
모든 스레드는 자신만의 커널 스택을 가진다.  

커널 스택은 대략 20KB 크기이며, User Stack과 마찬가지로 첫 1Page는 commit 해두고 그다음 Page는 guard page로 설정해둔다. 나머지는 reserve 상태이다.  
그리고 필요한 만큼 commit하여 사용한다.  
커널 스택의 크기가 작은 이유는, 커널 메모리영역은 OS와 모든 프로세스가 공유하는 영역이기 때문에 함부로 메모리를 많이 사용할 수 없기 때문이다.  

커널 스택이 User Stack과 분리되어 있는 이유는 보안상의 문제이다.  
User Stack은 사용자가 마음대로 값을 보고 조작할 수 있는데, 실수로 커널이 사용하는 메모리영역을 건드려서는 안되기 때문이다.  
그래서 커널모드에서는 커널스택을 따로두어 작업한다.
